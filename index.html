ðŸš€ Understanding Big Data & Analytics

Big Data refers to extremely large datasets that are difficult to process using traditional data processing applications. It's characterized by the three V's: Volume (large amounts of data), Velocity (fast data generation), and Variety (different data types). This portfolio showcases practical implementation of distributed computing frameworks designed to handle such massive datasets.

The Hadoop ecosystem provides a scalable solution for storing and processing data across distributed clusters. By implementing MapReduce, Hive, and Pig on top of HDFS, we can perform complex analytics on terabytes of data, enabling organizations to extract actionable insights and make data-driven decisions.

Hadoop Setup & WordCount

Installed and configured Hadoop on Windows, set up HDFS, and ran distributed MapReduce programs to process text files.

  • Configured HDFS directories and environment variables
  • Formatted NameNode and launched daemons
  • Validated via web interface at localhost:9870
hdfs dfs -mkdir /input
hdfs dfs -copyFromLocal input.txt /input/
hadoop jar hadoop-mapreduce-examples.jar \
  wordcount /input /output

Pig & Hive Analytics

Built data transformation pipelines with Pig Latin and executed SQL-like queries on Hadoop data using Hive.

  • Pig: Data filtering, grouping, and transformations
  • Hive: SQL queries, partitioning, and UDFs
-- Pig Latin
students = LOAD '/pigdata/students.csv' 
  USING PigStorage(',');
high_scorers = FILTER students BY marks > 80;

-- Hive SQL
CREATE TABLE students_text(
  id INT, name STRING, marks INT
);
LOAD DATA INPATH '/hivedata/students.csv' 
  INTO TABLE students_text;
